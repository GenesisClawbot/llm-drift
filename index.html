The Problem

LLMs update constantly. When they do, your prompts may stop working. Your JSON parser breaks. Your classifier returns different answers. Your code generator ignores format instructions.

You find out from your users. Not from the model provider.

How It Works

# pip install driftwatch from driftwatch import Monitor monitor = Monitor(model="gpt-4o", baseline="baseline_prompts.json") score = monitor.check_drift("test_prompts.json") if score > 0.15: send_alert("LLM drift detected: {:.2f}".format(score))

Run 20 standardized prompts every 6 hours. Compare outputs against baseline using cosine similarity. Get alerts when drift exceeds threshold.

Features

🔍 20 Test Prompts

Curated prompts across 7 categories: JSON, classification, summarization, code generation, and more.

📊 Similarity Scoring

Cosine similarity against baseline. Threshold alerts. Historical tracking.

🔔 Alerting

Email alerts when drift detected. Slack webhook support.

📈 Dashboard

Interactive demo. Visualize drift scores over time.

Real Drift Detected

We ran DriftWatch for 30 days. Here's what we caught:

  • Claude 3 Haiku added apologetic preambles to certain response types (Day 5, drift score: 0.23)
  • GPT-4o started adding "Based on the analysis..." prefixes to JSON responses (Day 12, drift score: 0.31)
  • Both models returned to baseline by Day 15 — proving drift can be temporary
