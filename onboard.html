You're in. Let's catch your first LLM drift.

Follow these 3 steps to have DriftWatch monitoring your endpoints in under 10 minutes.

1

Clone the repository & install

DriftWatch runs from your machine or CI/CD pipeline. Clone the repo and install the single dependency.

git clone https://github.com/GenesisClawbot/llm-drift.git cd llm-drift pip install -r requirements.txt
2

Set your API key & run baseline

Set your LLM API key and establish a baseline — this is what future runs are compared against.

# For Claude (Anthropic): export ANTHROPIC_API_KEY=sk-ant-... # Establish baseline (stores responses for all 20 test prompts) python3 core/drift_detector.py --run baseline # Output: data/baseline.json
💡 Run baseline when your model is behaving correctly — this becomes your reference point.
3

Run your first drift check

Run a drift check to see current deviation from baseline. Schedule this hourly in CI/CD for continuous monitoring.

# Single drift check python3 core/drift_detector.py --run check # Results: data/results.json + console output # Dashboard: open dashboard/index.html in your browser # ── Schedule hourly (GitHub Actions) ────────────────── # See .github/workflows/drift-check.yml in the repo. # Add ANTHROPIC_API_KEY to your repo secrets.

Set up GitHub Actions monitoring

The repo includes a pre-configured GitHub Actions workflow that runs drift checks every hour automatically. Just add your ANTHROPIC_API_KEY to your repo secrets.
